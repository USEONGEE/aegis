# Chat Session 삭제 - v0.5.12

## 문제 정의

### 현상
- ChatListScreen에 대화 세션을 삭제하는 기능이 없음
- useChatStore에 세션/메시지 데이터는 AsyncStorage에 영속되지만, 사용자가 직접 정리할 수단이 없음
- MAX_SESSIONS(50) 한도에 도달해야 오래된 세션이 자동 제거됨

### 원인
- 초기 구현에서 세션 생성/조회에 집중, 삭제 기능 미구현
- Store에 `deleteSessions` 액션 미존재 (인터페이스만 추가된 상태)

### 영향
- 사용자가 불필요한 대화를 정리할 수 없어 목록이 지저분해짐
- 테스트/실수로 생성된 빈 세션이 계속 남음
- cron 자동 실행 세션도 쌓여서 사용자 세션과 섞임

### 목표
- ChatListScreen에서 **편집 모드**를 통해 여러 세션을 선택 후 일괄 삭제
- 삭제 시 세션 메타데이터, 메시지, 스트림 커서를 함께 정리
- 현재 활성 세션이 삭제되면 currentSessionId를 null로 리셋

### 비목표 (Out of Scope)
- 서버(Relay/Redis) 측 세션 데이터 삭제 — 앱 로컬만 대상
- 개별 메시지 삭제 (세션 단위 삭제만)
- 세션 아카이브/숨김 기능
- 세션 검색/필터

## 제약사항
- RN 기본 컴포넌트만 사용 (외부 라이브러리 추가 없음)
- 기존 다크 테마 (#0a0a0a 배경) 스타일 유지
- 삭제 전 확인 Alert 필수 (되돌릴 수 없으므로)

## UX 결정사항
- **진입**: 헤더에 "편집" 버튼 → 선택 모드 진입
- **선택**: 각 세션 카드에 체크박스 표시
- **액션**: 하단에 "전체선택" + "삭제" 버튼
- **취소**: 헤더 "편집" → "완료" 토글로 선택 모드 해제
