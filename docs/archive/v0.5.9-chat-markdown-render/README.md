# Chat Markdown 렌더링 - v0.5.9

## 문제 정의

### 현상
- AI(assistant) 응답이 plain `<Text>` 컴포넌트로 렌더링됨
- OpenClaw/Claude가 반환하는 마크다운 문법(`**bold**`, `# heading`, `` `code` ``, 목록, 테이블 등)이 원시 텍스트 그대로 노출
- 코드 블록, 목록, 강조 등 구조화된 정보가 가독성 없이 표시됨

### 원인
- `ChatDetailScreen.tsx:397`에서 `<Text style={styles.messageText}>{item.content}</Text>`로 렌더링
- 마크다운 파싱/렌더링 라이브러리가 설치되어 있지 않음
- AI 응답의 콘텐츠 형식(markdown)과 렌더링 방식(plain text)의 불일치

### 영향
- AI가 구조화된 정보(포트폴리오, tx 결과, 정책 목록 등)를 보여줘도 읽기 어려움
- 코드 블록이 인라인으로 뭉개져 기술 응답의 가독성 저하
- DeFi 에이전트의 응답 품질 대비 표현 품질이 낮음

### 목표
- AI(assistant) 응답을 마크다운으로 렌더링하여 가독성 확보
- 헤딩, 볼드, 코드블록, 목록, 테이블 등 기본 마크다운 문법 지원
- 사용자(user) 메시지는 기존 plain text 유지

### 비목표 (Out of Scope)
- 마크다운 편집기 (사용자 입력에 마크다운 적용)
- 이미지/미디어 임베딩
- 커스텀 마크다운 확장 (수식, 차트 등)
- 스트리밍 중 실시간 마크다운 파싱 최적화 (후속 Phase에서 검토)

## 제약사항
- Expo 호환 필요 (네이티브 모듈 의존성 최소화)
- 스트리밍 토큰 누적 시 리렌더링 성능 고려 필요
- 기존 `styles.messageText` 톤과 조화되는 마크다운 스타일
