## 6하원칙

### Who (누가)
- 다음 세션 / 어떤 agent든
- 필요 권한: 파일 시스템 (packages/app, packages/daemon, packages/openclaw-plugin, 루트 package.json)

### What (무엇을)

#### HIGH — App 상태 정합성 (v0.5.8 + v0.5.12)
- [ ] **#1** non-current 세션의 done/error/cancelled 이벤트에서 transient reset 누락 → isLoading 고착
  - `ChatDetailScreen.tsx` L160, L215, L236: `isCurrentSession`일 때만 reset → eventData.sessionId 기준으로 변경
- [ ] **#2** streamBufferRef가 컴포넌트 로컬 ref → 세션 전환 시 스트림 내용 유실
  - `ChatDetailScreen.tsx` L57: streamBufferRef/streamMsgIdRef → store sessionTransient로 이동 또는 done 시 전체 메시지 보정
- [ ] **#3** late event로 삭제된 세션 재생성 방어 (v0.5.8 + v0.5.12 공통)
  - `useChatStore.ts`: addMessage(), setSessionTransient(), updateCursor()가 없는 세션을 자동 생성
  - 방향: `deletedSessionIds` persisted tombstone + ingress에서 tombstoned session 무시

#### HIGH — Markdown 보안/성능 (v0.5.9)
- [ ] **#4** 이미지 렌더러: 스타일 숨김만 → FitImage가 여전히 remote URI에 네트워크 요청
  - `markdownDarkStyles.ts` L119: 스타일이 아닌 renderRules에서 image를 null 반환
- [ ] **#5** 스트리밍 중 매 delta마다 마크다운 재파싱 + persist + auto-scroll
  - 방향: 스트리밍 중 plain text → done 시 마크다운 렌더링, 또는 throttle/debounce

#### HIGH — Wallet 상태 정합성 (v0.5.10)
- [ ] **#7** 지갑 삭제 후 selectedAccountIndex=0 폴백 → 0번 삭제 시 깨짐
  - `DashboardScreen.tsx` L239, `useWalletStore.ts` L45: 남은 지갑 중 첫 번째로 폴백 + addresses 정리
- [ ] **#8** fetchPortfolio race condition — 늦은 응답이 현재 지갑 포트폴리오 덮어씀
  - 방향: 요청 시점 accountIndex를 응답과 매칭 또는 AbortController

#### HIGH — Daemon 도구 연결 (v0.5.11)
- [ ] **#9** LLM-facing 도구 설명에 policyRequest 선행 플로우 누락
  - `ai-tool-schema.ts` L237, `index.ts` L157, `TOOLS.md` L41/L81: "policyRequest 선행 후 sendTransaction" 명시
- [ ] **#10** manifest dynamic import: daemon package.json에 @wdk-app/manifest 의존성 미선언
  - `tool-surface.ts` L586: ERR_MODULE_NOT_FOUND 재현됨
  - 방향: daemon dependencies에 선언 + manifest 패키지 빌드 출력 설정

#### HIGH — 인프라 잔존 (v0.5.13)
- [ ] **#11** 루트 package.json workspaces에 kitten-cli 잔존 + package-lock.json 엔트리 잔존
  - 방향: workspaces에서 제거 → npm install로 lockfile 재생성

#### MED
- [ ] **#6** 링크 보안: tel:/sms:/intent:// 등 deep-link scheme 허용 + linkify: false로 bare URL 클릭 불가
  - 방향: allowlist 스킴(https, http) + linkify: true
- [ ] **#12** 도구 개수 불일치: 문서 15개 vs 테스트 16개 vs TOOLS.md 16개
  - 방향: 실제 등록 도구 수 확인 후 통일

### Why (왜)
- **#1~3**: 멀티세션 UX의 근본 결함. 세션 전환/삭제 시 상태 오염 → 사용자에게 고착된 로딩, 유령 세션 노출
- **#4~5**: 보안(트래킹 픽셀) + 성능(긴 응답 버벅임) → 데모/실사용 모두 영향
- **#7~8**: 지갑 UI 신뢰성 → 잘못된 잔액 표시는 DeFi 앱에서 치명적
- **#9~10**: manifest 도구가 사실상 사용 불가 상태 → v0.5.11의 핵심 목표 미달
- **#11**: npm install 시 경고/에러 가능성
